朋友圈如何发送无损视频？
2024-07-19

今天无意间发现了微信朋友圈如何发送无损视频的方法，在之前其实有观察到一些好友分享朋友圈的视频帧率在 60Hz+，但一直没去了解。下面结合我以往的开发经验以及实际的测试结果分享朋友圈如何发送无损视频：

背景及压缩原因

每次不通过视频号分享朋友圈的视频我一般都是先剪辑完成品后直接上传分享，但这样的话分享的视频是走默认通道存储在微信的“性能云端库”（我随便起的名字，意思就是默认分享的视频会压缩画面和帧率从而达到节省内存和提升视频首次加载速度的目的，音频方面未测试压缩比），视频帧率经过压缩后浮动范围在 28~30Hz，同时呢默认通道限制视频时长在 30s 以内。以下面的视频为例，压缩帧率从 60Hz 变为 28+Hz，画面明显压缩（具体视频画质参数未获取，懒🥱）

解决方案

如果我们需要发送原画质视频需要在选取视频的界面下面勾选✔ 制作视频，它会在允许发送原画质视频的基础上进行剪辑操作，最终被允许发送 60s 内的原画质二次创作视频（当然你不进行二次剪辑就相当于是原视频咯），但它会给你增加一些源信息导致你视频内存变大（上述视频走原画质通道 视频从 24.2MB 变为 38MB，而走默认通道的话则从 24.2MB 压缩至 2.1MB）。最后无论是默认通道还是原画质通道都会保存视频备份到本地。

原画质通道和默认通道是我虚构的概念，只是为了方便理解，其本质还是调用各自视频分类处理的 API
